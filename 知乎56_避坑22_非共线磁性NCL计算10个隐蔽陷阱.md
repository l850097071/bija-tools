## 为什么非共线计算比你以为的更容易翻车？

非共线磁性(NCL)加自旋轨道耦合(SOC)是DFT计算的"高端操作"——涉及磁各向异性(MAE)、Dzyaloshinskii-Moriya相互作用(DMI)、拓扑磁结构时,NCL+SOC是必选项。但问题是:VASP的NCL模式自6.5.0以来进入了剧烈变化期,多个历史兼容行为被移除,同时旧bug暴露、新边界条件出现。

很多课题组还在用3年前的INCAR模板跑NCL——这套模板在新版本上要么直接崩溃,要么静默产出错误结果。

本篇覆盖的10个陷阱,前3个是**让你直接crash的致命错误**,中间4个是**不报错但结果有系统性偏差的隐蔽bug**,末3个是**2025-2026年新暴露的边界条件**。

## 陷阱#1: ISPIN=2 + LNONCOLLINEAR → 直接崩溃 (VASP ≥ 6.5.0)

**严重级别**: 🔴 致命（代码直接crash,无结果产出）

**症状**: VASP 6.5.0及以上版本,INCAR中同时设置`ISPIN=2`和`LNONCOLLINEAR=.TRUE.`,代码**蓄意崩溃**。错误信息提示:不允许此组合。

**根因**: 这是VASP开发者在2025年1月7日引入的**破坏性变更**(breaking change)。历史版本中,VASP遇到`ISPIN=2 + LNONCOLLINEAR=.TRUE.`时内部静默覆写为`ISPIN=1`,但这个fallback逻辑被移除了。开发者的理由是:让用户"明确意识到自己在做什么"——非共线计算不应设置ISPIN。

**修复**: 
- 显式设置`ISPIN = 1`,或者完全不写ISPIN这一行
- 非共线+自旋轨道耦合的磁学行为完全由`LNONCOLLINEAR`和`LSORBIT`控制
- 如果你在实验不同的磁基态,用`MAGMOM`的初始猜测值去探索,而不是ISPIN

**你的检查清单**:
- [ ] INCAR中是否出现了`ISPIN = 2`？
- [ ] 同时是否有`LNONCOLLINEAR = .TRUE.`或`LSORBIT = .TRUE.`？
- [ ] 如果VASP版本≥6.5.0,上述组合=一定crash

## 陷阱#2: 用错可执行文件——vasp_std 跑不动 NCL

**严重级别**: 🔴 致命（代码crash或报错退出）

**症状**: 提交作业后VASP报错:"non collinear calculations require that VASP is compiled without the flag -DNGXhalf and -DNGZhalf"。新手常见的反应是去改编译选项——实际是你用了错误的二进制文件。

**根因**: VASP编译时生成多个可执行文件:
- `vasp_std`: 标准版,不支持非共线（用于常规collinear计算）
- `vasp_ncl`: 非共线版,**唯一**支持NCL+SOC的版本
- `vasp_gam`: Gamma点优化版,同样不支持NCL

`vasp_std`和`vasp_gam`在编译时启用了`-DNGXhalf`和`-DNGZhalf`标志——这些标志与NCL的旋量波函数存储格式不兼容。

**修复**:
- 检查你的作业提交脚本(pbs/slurm/sbatch),确认调用的可执行文件是`vasp_ncl`
- 不要试图用`vasp_std`+特殊INCAR设置来绕过——不存在这样的设置
- 如果集群上没有`vasp_ncl`——联系管理员编译一个

**你的检查清单**:
- [ ] `which vasp_ncl` 或作业脚本中的`EXEC`变量指向`vasp_ncl`？
- [ ] 不是`vasp_std`,不是`vasp_gam`
- [ ] 编译时的`-DNGXhalf/-DNGZhalf`标志禁用了

## 陷阱#3: SAXIS + LDA+U + WAVECAR重启 → 伪局部极小值

**严重级别**: 🔴 隐蔽——不crash,但给错误能量（违反时间反演对称性）

**症状**: 你在做MAE计算,用一个方向的SAXIS(如`SAXIS = 0 0 1`)收敛后,从WAVECAR重启,改变SAXIS(如改成`SAXIS = 0 0 -1`)。结果`SAXIS = 0 0 1`和`SAXIS = 0 0 -1`给出**不同的总能量**——这在物理上是不可能的(时间反演对称性要求这两个方向的能量相等)。

**根因**: 2025年1月,VASP论坛用户发现:在FePS₃体系中,当LDA+U与SOC同时激活,从WAVECAR重启并改变SAXIS时,U张量施加的轨道极化与SOC的SAXIS旋转产生竞争,系统被锁定在一个错误的局部极小值上。

更具体地说:Dudarev的LDA+U(以及Liechtenstein的完整U-J形式)通过占据矩阵重塑特定轨道的局域态密度,这种重塑方向与SOC的量化轴方向在WAVECAR中形成了一种**不可逆的耦合**。改变SAXIS时,U惩罚项的方向没有同时更新。

**修复**:
- 每个SAXIS方向从**零开始计算**: `LWAVE = .FALSE.` + `LCHARG = .FALSE.`
- 替代方案:保持SAXIS不变,改为旋转`MAGMOM`的初始方向——MAGMOM在SCF过程中可自由演化,不会像SAXIS那样产生硬约束
- VASP团队已确认这个问题,并计划在wiki中添加警告——但截至2025年底仍是open issue

**你的检查清单**:
- [ ] 你是否在做多SAXIS方向的MAE计算？
- [ ] 是否使用了LDA+U？
- [ ] 如果两个都是YES——每个SAXIS方向是否从`ISTART=0`开始？
- [ ] 是否尝试过改MAGMOM来替代改SAXIS？

## 陷阱#4: MAGMOM初始猜测敏感——能量差可达50 meV

**严重级别**: 🟡 不crash,但不同初始猜测→不同终态→论文可能选到错误基态

**症状**: 你用`MAGMOM = 1 1 1`跑NCL+SOC收敛到一个磁构型,同行用`MAGMOM = 0.5 0.5 0.5`跑**同一个体系**,得到能量差50 meV,磁矩方向完全不同。谁是对的？

**根因**: 这不是bug——这是NCL能量面的内在属性。NCL+SOC的能量函数是`E = E[n, m_x, m_y, m_z]`,其中四个自由度(电荷密度+三方向磁化矢量)构成一个高维能量面。这个能量面可以存在**多个局部极小值**,它们之间能量差很小,但磁化方向完全不同——都是物理上合法的解。

2025年6月,simon_cao在VASP论坛报告:改变MAGMOM初始值导致最终总能量差异达50 meV,磁矩矢量完全重新排列。这不是计算错误——是**你的体系天然具有多个近简并的磁基态**。

**修复**:
- 永远不要只用一个MAGMOM猜测值——至少测试3种: `MAGMOM = 0 0 0`(非磁起点)、`MAGMOM = 1 1 1`(共线猜测)、`MAGMOM = 1 0 0`(面内猜测)
- 从非磁WAVECAR开始(`ISTART=0`),然后分别测试几种不同的MAGMOM初始磁矩排列
- 能量最低的构型选为基态——但要在论文中说明"在测试的X种初始磁构型中..."
- 考虑用非共线的MAGMOM指定更复杂的初始图案:`MAGMOM = N*1.0 0.0 0.0 N*0.0 1.0 0.0`(反铁磁非共线试探)

**你的检查清单**:
- [ ] 是否测试了≥3种不同的MAGMOM初始值？
- [ ] 是否从非磁起点(ISTART=0)开始了至少一组测试？
- [ ] 论文中是否声明了磁基态搜索的方法和局限性？

## 陷阱#5: 对称性bug——DFPT导数静默错误

**严重级别**: 🔴 不crash,但介电张量和Born有效电荷是错的

**症状**: 你跑NCL+SOC弛豫正常,加`LEPSILON=.TRUE.`算介电张量和Born有效电荷,结果看起来合理——但数值有一个系统性的错误分量,在你对比实验红外谱时才发现不对。

**根因**: VASP中NCL模式与对称性分析的交互存在**已知的且尚未修复的bug**。当`ISYM`不为-1时(即VASP自动检测对称性),NCL+SOC的DFPT响应函数计算会产生错误的介电张量分量和Born有效电荷。这个bug在VASP < 6.5.0中确认存在。

本质上:NCL模式下的自旋密度矩阵变换规律与标准DFPT的对称性约化假设不一致——VASP的对称性分析模块没有完全适配NCL的旋量结构。

**修复**:
- 所有NCL生产计算中使用`ISYM = -1`或`ISYM = 0`——完全禁用对称性
- 代价:计算量增加(不可约k点变为全BZ)——但**换来了数值可靠性**
- Wannier90接口也有相同问题(见陷阱#9)

**你的检查清单**:
- [ ] INCAR中有`ISYM = -1`或`ISYM = 0`吗？
- [ ] 如果你在用`ISYM = 2`(默认)——你的介电张量/Born电荷可能有问题
- [ ] 如果ISYM=0导致k点过多——至少用ISYM=0跑一次验证,对比ISYM=2的结果

## 陷阱#6: 缺少三个"强烈推荐"标签——精度悄悄丢失

**严重级别**: 🟡 不crash,但精度系统性偏低

**症状**: 你的NCL+SOC计算正常收敛,但磁矩总比别人小一点、轨道矩差4-5倍、MAE精度受质疑。

**根因**: NCL计算对数值精度要求高于collinear计算——旋量波函数的两个分量间的干涉需要更高的积分精度。但很多INCAR模板只有`LNONCOLLINEAR = .TRUE.`和`LSORBIT = .TRUE.`,缺少三个关键的精度增强标签:

1. **`GGA_COMPAT = .FALSE.`** — 使用更精确的GGA数值积分方案。默认值`.TRUE.`是VASP的历史遗留(为了与旧版PW91兼容),对大多数GGA泛函(PBE/RPBE/PBEsol)应该关闭
2. **`LASPH = .TRUE.`** — 在PAW球内计入非球对称的电子密度贡献。在SOC存在时,自旋密度在PAW球内天然非球对称——不开启此项等于系统性地丢弃了一部分自旋密度
3. **`ISYM = 0`** — 如陷阱#5所述,对称性与NCL/SOC存在已知bug

**修复**:
```fortran
# 所有NCL+SOC计算的INCAR"三件套":
GGA_COMPAT = .FALSE.
LASPH = .TRUE.
ISYM = 0
```
这三个标签**对任何NCL+SOC生产计算都是强烈推荐的**,VASP官方文档和论坛反复强调。但很多公开的INCAR模板里仍然缺它们。

**你的检查清单**:
- [ ] `GGA_COMPAT`是`.FALSE.`吗？（默认是`.TRUE.`）
- [ ] `LASPH`是`.TRUE.`吗？（默认是`.FALSE.`）
- [ ] `ISYM`不是`2`吗？（默认是`2`,即自动检测对称性）

## 陷阱#7: 两步法MAE → 轨道矩系统性低估4-5倍

**严重级别**: 🟡 不crash,MAE值可能接近但轨道矩数值错误

**症状**: 你按标准MAE协议操作:先用collinear跑SCF→不用SOC→收敛后用`ICHARG=11`+`vasp_ncl`加SOC做非自洽计算→得到MAE。结果:MAE值看起来合理(比如几百μeV量级),但当你汇报"Co的轨道矩为0.25 μB"时,审稿人说"这个值应该是1 μB左右"——你被问住了。

**根因**: 2025年9-11月VASP论坛的集中讨论发现,两步法(Collinear SCF → 非自洽SOC)会系统性地低估轨道矩约4-5倍。因为collinear计算产出的电荷密度中**没有SOC诱导的轨道极化信息**——而非自洽SOC计算(`ICHARG=11`)只读取电荷密度,无法恢复这部分轨道信息。

虽然MAE(总能量差)的值通过力定理(force theorem)通常能部分恢复,但轨道矩的绝对值受此影响显著。如果你的论文在讨论"轨道矩对MAE的贡献"(如Bruno模型),两步法的轨道矩数值会严重误导结论。

**修复**:
- 全自洽SOC计算:`vasp_ncl` + SOC从头SCF(ISTART=0),不经过collinear步骤
- 或者用`ALGO = All`(一步对角化,不迭代),报告称对某些体系可以解决轨道矩问题
- 如果计算资源不够跑全自洽SOC——至少**在论文中声明这是non-self-consistent SOC**,轨道矩可能存在低估

**你的检查清单**:
- [ ] MAE是怎么算的？两步法(Collinear→非自洽SOC)还是全自洽SOC？
- [ ] 如果你在讨论轨道矩贡献——数值来自两步法吗？如果是,标明了"非自洽"吗？
- [ ] 有没有至少一个关键方向用全自洽SOC验证过？

## 陷阱#8: 约束磁矩的惩罚能量污染 → MAE数值不可用

**严重级别**: 🔴 数值污染——直接导致错误结论

**症状**: 你开启了`I_CONSTRAINED_M`来约束磁矩方向(比如在做MAE计算时"把自旋钉在z方向"),然后直接读OSZICAR里的总能量做MAE分析。你得到"MAE = 1.2 meV"并写进论文——但这是错的。

**根因**: `I_CONSTRAINED_M`在总能量上加了一个**惩罚项**(penalty energy),其形式是`E_penalty = λ * Σ |M_i - M_target|²`。关键是——这个惩罚项的大小没有物理意义,纯粹是数值拉格朗日乘子。而且当你改变约束方向时,惩罚项的大小也变了——所以两个方向的`E_total - E_penalty`差才是真正的MAE,而不是直接用总能量差。

这个错误在VASP社区被反复纠正,但每年都有新论文犯。

**修复**:
- **MAE计算绝不用约束磁矩**——MAE的精度要求μeV级别,惩罚项的能量量级远大于MAE本身
- 约束磁矩仅用于**收敛辅助**:先约束到目标构型→收敛→去掉约束→再做自由SCF→读自由SCF的能量做MAE
- 两步法流程: (1) `I_CONSTRAINED_M`收敛到目标方向 → (2) 去掉约束+保持MAGMOM不变+继续SCF → (3) 用第二步的能量做MAE

**你的检查清单**:
- [ ] INCAR中是否有`I_CONSTRAINED_M`？
- [ ] 如果有——你是在用它做收敛辅助,还是直接读它的总能量？
- [ ] MAE分析用的是约束态能量还是自由态能量？

## 陷阱#9: Wannier90接口——NCL模式下AMN/MMN静默错误

**严重级别**: 🔴 不crash,但Wannier插值能带是错的

**症状**: NCL+SOC计算成功后,调用Wannier90做紧束缚拟合和能带插值,Wannier90正常输出`wannier90.bxsf`。你用Xmgrace/VESTA画出来——看起来和DFT能带"差不多"——但有一个自旋分量被静默地错误投影了。

**根因**: VASP的Wannier90接口代码`mlwf.F`中存在spin axis处理bug。对于非共线计算,`mlwf.F`在读取投影的spin quantization axis时,错误地只取了第三个分量而不是完整的三分量矢量:

```fortran
! Buggy line in mlwf.F (known issue as of VASP 6.4.3):
MLWF%LPRJ_functions(IS)%spin_qaxis = proj_s_qaxisx(3,IS)  
! Should be:
MLWF%LPRJ_functions(IS)%spin_qaxis = proj_s_qaxisx(:,IS)
```

结果:Wannier90得到的自旋量子化轴是错误的→紧束缚矩阵`AMN`/`MMN`的自旋分量错误→最后的Wannier插值能带可能是错的,特别是自旋分辨的能带。

**修复**:
- 使用`ISYM = -1`作为workaround(禁对称性避免进一步复杂化spin axis)
- 确保VASP和Wannier90都是最新版本——这个bug在某些版本中已修复(但VASP 6.4.3中确认存在)
- 如果必须用Wannier90+NCL——对比至少几个高对称k点的DFT能带和Wannier插值能带,确认一致

**你的检查清单**:
- [ ] 是否使用了Wannier90 + NCL？
- [ ] 是否对比了DFT能带和Wannier插值能带？
- [ ] 自旋分辨的能带是否与DFT一致？

## 陷阱#10: SAXIS的物理含义和限制——它不是你想的那个"旋转磁矩的把手"

**严重级别**: 🟡 概念错误,导致INCAR设置和物理解释都出问题

**症状**: 你设置`SAXIS = 0 1 0`想"让磁矩指向y方向"——但VASP并没有把你的磁矩"旋转"到y方向。你困惑:为什么MAGMOM一开始指向z,加SAXIS后磁矩方向似乎没完全变到y？

**根因**: `SAXIS`的作用比很多人以为的要窄——它**只控制SOC项中的自旋量子化轴方向**,不直接影响MAGMOM的初始方向,不影响交换关联泛函,也不影响LDA+U的占位矩阵方向。

具体来说:
- SOC Hamiltonian的`ξ L·S`项中的`S`算符是在SAXIS定义的量子化轴下计算的
- 但交换关联的`B_xc`场作用在自旋密度上,自旋密度由MAGMOM的演化决定——不是由SAXIS强制的
- LDA+U的占位矩阵有自己的局域坐标——与SAXIS无关

因此,如果你想要磁矩指向[110]方向——仅仅设置`SAXIS = 1 1 0`是不够的,你还需要让MAGMOM的初始猜测也指向[110],然后让SCF把它自然地收敛到那个方向。

**修复**:
- 对于MAE计算:固定MAGMOM方向为几种高对称方向,分别改变SAXIS来评估能量差——这是标准做法
- 对于磁基态搜索:不要依赖SAXIS"旋转"磁矩——使用不同方向的MAGMOM初始猜测
- SAXIS + MAGMOM要协调:如果你设置SAXIS指向z,MAGMOM也应该以z方向为主(除非你刻意在探索非共线基态)

**你的检查清单**:
- [ ] SAXIS的作用是否理解正确?（只影响SOC,不强制磁矩方向）
- [ ] MAGMOM是否与SAXIS协调?
- [ ] LDA+U体系是否考虑了U与SAXIS的交互(见陷阱#3)？

## NCL+SOC计算终极检查清单

在提交你的NCL+SOC作业之前,逐项检查:

### 致命错误防护（crash级）
- [ ] VASP版本≥6.5.0 → `ISPIN = 1`(或不写)
- [ ] 可执行文件是`vasp_ncl`,不是`vasp_std`或`vasp_gam`

### 精度与可靠性
- [ ] `GGA_COMPAT = .FALSE.` ✅
- [ ] `LASPH = .TRUE.` ✅
- [ ] `ISYM = -1` 或 `ISYM = 0` ✅
- [ ] 测试了≥3种MAGMOM初始值
- [ ] 如果用了LDA+U + SAXIS → 每个SAXIS方向从零开始

### MAE计算专项
- [ ] 没有用`I_CONSTRAINED_M`直接产出的能量做MAE
- [ ] 轨道矩是否标注了全自洽/非自洽？
- [ ] 是否至少有一个方向用全自洽SOC验证？

### Wannier90专项
