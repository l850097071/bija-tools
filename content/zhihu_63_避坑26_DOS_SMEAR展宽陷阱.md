# 知乎63: 避坑#26——态密度DOS SMEAR/展宽：你以为在画DOS，其实在画高斯函数

**系列**: 避坑系列#26 | **字数**: ~4200 | **定位**: 催化/DFT计算化学垂直·真实经验

---

上一期讲了能带结构的K点路径陷阱，评论区有人问："能带和DOS是一家的，DOS有没有类似的坑？"

有。而且DOS的坑比能带更隐蔽——能带的错至少画出来形状不对，你能看出来。DOS的错画出来形状"看起来很好"，但物理全错。你贴在论文里，审稿人如果懂行，一眼就能判你死刑。

今天专讲DOS计算里最被低估的杀手：SMEAR/展宽。

---

## 陷阱1: SCF的SIGMA直接搬到DOS——你画的是高斯函数，不是态密度

**死法**: SCF用ISMEAR=0, SIGMA=0.2跑完收敛。ICHARG=11做DOS，不改ISMEAR和SIGMA。

**物理原因**: ISMEAR=0 (Gaussian smearing)在费米能级附近引入了一个高斯展宽函数，宽度=σ=SIGMA。你用SIGMA=0.2eV跑SCF——OK，电子在0.2eV的能量窗口内被部分占据，帮助收敛。但**DOS是把占据数对能量求导**。你用SIGMA=0.2跑DOS→DOS(E) = Σ δ(E-ε_i) * G(σ=0.2)，你看到的是高斯卷积的结果，不是真实的δ函数态密度。

**审稿人眼里**: 费米能级附近的峰值宽度≈2σ=0.4eV。如果你的d带中心偏移0.2eV（这在催化里是显著差异），你的DOS根本分辨不了——因为展宽本身就0.4eV。

**真实案例**: 一篇投Nature Catalysis的稿子，Co-N₄位点DOS在费米能级处有个"尖峰"，作者解读为"高反应活性的电子结构特征"。审稿人问了一句："你DOS用的什么SIGMA？" 作者答0.2。审稿人直接在回复里写："Replace SIGMA=0.2 with ISMEAR=-5 and the 'sharp peak' disappears." 拒稿。

**操作铁律**:
```
SCF: ISMEAR=0, SIGMA=0.05-0.1  ← 帮助收敛，允许
DOS: ISMEAR=-5 (Blochl tetrahedron) ← 零展宽，物理DOS
     如果体系有带隙: ISMEAR=0, SIGMA ≤ 0.05 ← 勉强可接受
     金属体系: ISMEAR=-5 强制，或 ISMEAR=1, SIGMA ≤ 0.05
```

**为什么不能都用ISMEAR=-5**: tetrahedron方法需要足够密的K点（≥4个不可约K点）。K点太稀→tetrahedron产生负DOS（非物理）。此时退而求其次用ISMEAR=0 + SIGMA≤0.05。

## 陷阱2: SIGMA > kBT 时，你引入的能量误差比你计算的吸附能还大

**死法**: SIGMA设太大（比如0.3-0.5eV），跑完报告总能量，去做吸附能。

**物理原因**: 展宽引入的"熵项"（entropy term） = -T*S，在OSZICAR里显示。T*S = σ × Σ f_i ln(f_i) + (1-f_i) ln(1-f_i)。如果SIGMA太大，这个熵项可以大到0.1-0.5 eV/原子。对于吸附能（通常0.1-0.5 eV），你的展宽误差和你要计算的物理量一个数量级。

**量化**: 
- SIGMA=0.05 → 熵项 ~0.001-0.01 eV → 安全
- SIGMA=0.20 → 熵项 ~0.01-0.05 eV → 临界
- SIGMA=0.50 → 熵项 ~0.05-0.20 eV → 不可接受（吸附能级别误差）

**操作铁律**: OSZICAR里看 `EENTRO` 这一项。如果EENTRO > 0.001 eV/atom → SIGMA太大，降低。

**进阶技巧**: 对于高精度能量计算（吸附能/形成能/反应能），用 `ISMEAR=-5` 跑最终静态计算（NSW=0, ICHARG=0从SCF的CHGCAR读入），这是零展宽能量，没有熵项污染。

## 陷阱3: ISMEAR=-1 (Fermi-Dirac) 的物理意义被你忽略了

**死法**: 用ISMEAR=-1跑SCF，不设置SIGMA（默认0.01），然后直接做能量分析。

**物理原因**: ISMEAR=-1是真实的Fermi-Dirac分布——它模拟了有限温度下电子的真实占据。这**不是数值技巧**，它是物理的。SIGMA=k_B T。默认SIGMA=0.01 eV ≈ 116K。你的计算在116K下进行。对于室温（300K）→ SIGMA对应约0.026 eV。

**什么时候用ISMEAR=-1**: 
- 有带隙的体系（半导体/绝缘体）——Fermi-Dirac比Gaussian更物理
- 需要温度效应时（如计算自由能随温度变化）
- **禁止用于金属体系**——Fermi-Dirac在费米能级处导数有问题，金属DOS会失真

**操作铁律**:
```
半导体: ISMEAR=-1, SIGMA=0.01 → 可接受
         ISMEAR=0, SIGMA=0.05 → 也行（Gaussian更稳定收敛）
金属:   ISMEAR=1 (MP, 一阶) 或 ISMEAR=0 (Gaussian, 零阶)
         ISMEAR=-5 (tetrahedron) → 最佳但需密K点
```

## 陷阱4: Methfessel-Paxton (ISMEAR=1) 的负占据——不是bug，是feature，但你得知道它存在

**死法**: 用ISMEAR=1跑金属体系，在DOS里看到费米能级以上有"负的DOS"或者占据数>1的部分，以为计算出错。

**物理原因**: MP展宽（ISMEAR=1）是Gaussian展宽的一阶修正——它在能量上引入一个多项式修正，使得展宽后的总能量更接近零展宽极限。代价是：占据数f(E)可以略小于0或略大于1。这是**故意的数学构造**，不是bug。MP方法的总能量外推到σ→0时比Gaussian更快收敛。

**但这有个坑**: 如果你直接看PROCAR或用占据数做Bader电荷分析——占据数<0或>1会造成非物理的电荷分配。

**操作铁律**:
```
MP展宽 (ISMEAR=1):
  ✅ 金属体系SCF收敛 — 总能量更准
  ✅ 力的计算 — 力的展宽外推比Gaussian好
  ❌ DOS直接画图 — f(E)可以>1或<0，DOS有artifact
  ❌ Bader/电荷分析 — 占据数非物理
  ❌ 能带结构 — 同上

解决: SCF用ISMEAR=1收敛 → 保存CHGCAR → 静态计算用ISMEAR=-5读取CHGCAR → 用静态计算的结果做DOS/电荷分析
```

## 陷阱5: Tetrahedron (ISMEAR=-5) 的K点密度诅咒

**死法**: 看到手册说"ISMEAR=-5最准确"，兴冲冲地把所有计算都改成ISMEAR=-5，K点用的是以前2×2×2那套。

**物理原因**: Tetrahedron方法把布里渊区划分为四面体，在每个四面体内线性插值。如果K点太稀，四面体太大→插值误差>展宽误差。具体来说：
- 如果K点间距>能带曲率→tetrahedron在带边附近产生鬼峰/负DOS
- VASP会静默地输出这些非物理结果，不报错不警告

**K点密度门槛**:
- 半导体（能带平坦）: ≥ 4×4×4 → ISMEAR=-5 可
- 金属（能带陡峭）: ≥ 8×8×8 或更密 → ISMEAR=-5 可
- 任何布里渊区尺寸<4个不可约K点 → **禁止ISMEAR=-5**

**操作铁律**: 如果你不确定K点够不够密，做这个小测试：
1. 同一体系用ISMEAR=0, SIGMA=0.01 跑DOS
2. 同一体系用ISMEAR=-5跑DOS
3. 如果两条DOS曲线在费米能级附近对不上 → K点不够密，tetrahedron不可靠

## 陷阱6: 半导体用金属展宽——SIGMA吃掉带隙

**死法**: 体系有0.5eV带隙，但INCAR里用的ISMEAR=0, SIGMA=0.2（从金属体系的模板复制的）。

**物理原因**: Gaussian展宽把占据态"拖"到导带里，把空态"拖"到价带里。如果SIGMA和带隙可比（SIGMA=0.2 vs Gap=0.5），展宽会把带隙"模糊"掉40%（2σ=0.4eV ≈ 80%的带隙）。DOS图上价带顶和导带底之间的空隙消失或变小→你以为带隙就是展宽后的值（~0.3eV），实际上真实带隙是0.5eV。

**审稿人视角**: "Your DOS shows a band gap of 0.3 eV, but the band structure shows 0.5 eV. Explain." 你解释不了——因为这就是个bug。

**操作铁律**:
```
带隙 > 0.5 eV → ISMEAR=-1 或 -5, SIGMA无关（反正没态在费米能级处）
带隙 < 0.5 eV → ISMEAR=0, SIGMA ≤ 0.01（小带隙体系需要极小展宽）
金属/零带隙 → ISMEAR=0, SIGMA=0.05-0.1（收敛）→ 最终用ISMEAR=-5（准确）
```

## 陷阱7: 自旋极化体系——每个自旋通道的展宽独立施加，你忘了检查

**死法**: ISPIN=2计算，只看总DOS，不检查每个自旋通道。某自旋通道在费米能级附近"看起来是金属"但实际上是展宽造成的。

**物理原因**: 对于磁性体系，两个自旋通道的能带结构不同。可能spin-up是金属（态密度在费米能级非零），spin-down是半导体（有带隙）。如果你统一用ISMEAR=0, SIGMA=0.1——spin-down通道本来的带隙被展宽"填满"了，DOS图上看起来spin-down也是金属。

**后果**: 你的自旋极化度P = (N↑ - N↓)/(N↑ + N↓) 在费米能级处算出来是错的。

**操作铁律**:
1. 分开画spin-up和spin-down的DOS
2. 分别检查每个通道在费米能级附近是否有真的态密度还是展宽artifact
3. 如果某通道有带隙→该通道的展宽必须考虑带隙大小

---

## 陷阱8: NCL非共线磁性——SMEAR在自旋混合态下的隐藏行为

**死法**: 用LNONCOLLINEAR=TRUE跑SOC计算，ISMEAR沿用共线磁性的设置。

**物理原因**: NCL计算中每个K点的每个能带有一个2×2密度矩阵（而非一个标量占据数f）。展宽对这个2×2矩阵的每个分量独立作用。如果SIGMA设得大，自旋混合态的非对角分量也被展宽污染→自旋纹理(spin texture)分析、自旋霍尔电导等依赖自旋分量的物理量都会出错。

**操作铁律**:
```
NCL+SOC → ISMEAR=-5 或 ISMEAR=0 + SIGMA ≤ 0.02
自旋纹理分析 → 必须用 ISMEAR=-5 的静态计算
```

## 陷阱9: SIGMA的单位陷阱——你以为是eV，实际是...

**死法**: 不同代码的SMEAR/展宽参数单位不同。VASP里SIGMA的单位是eV。Quantum ESPRESSO里degauss的单位是Ry。Wien2k里用mRy。你从一篇文献里看到"smearing=0.01"，搬过来直接用——如果那篇文献用的是QE，0.01 Ry = 0.136 eV，差了13.6倍。

**常见混淆**:
| 代码 | 参数名 | 单位 | 0.1的物理含义 |
|------|--------|------|--------------|
| VASP | SIGMA | eV | σ=0.1 eV |
| QE | degauss | Ry | σ=0.1 Ry=1.36 eV |
| CP2K | SMEAR | Hartree | σ=0.1 Ha=2.72 eV |
| CASTEP | smearing_width | eV | σ=0.1 eV |
| Wien2k | TETRA/Gauss | mRy | σ=2 mRy=0.027 eV |

**操作铁律**: 从文献/教程里看到smearing参数→第一件事是查代码文档确认单位。不确认=闭眼下刀。

## 陷阱10: 最后但最致命的——你从来没检查过展宽到底在做什么

**死法**: INCAR里的ISMEAR和SIGMA是你从师兄的模板复制的，你从来没改过，也从来没看过OSZICAR里的entropy项。

这是所有陷阱的根源。

**一个自查**: 打开你最近跑的任何一个计算。看一眼OSZICAR里的 `EENTRO` 这一列。如果它的绝对值 > 0.005 eV/atom → 你的SIGMA太大了。如果你发现你从来不知道EENTRO在哪看 → 你踩过第1-9个陷阱的概率>80%。

**操作铁律 — DOS SMEAR终极检查单**:
```
[ ] SCF的SIGMA和DOS的SIGMA不同？
[ ] DOS用 ISMEAR=-5（tetrahedron）或 ISMEAR=0 + SIGMA≤0.05？
[ ] OSZICAR的 EENTRO < 0.001 eV/atom？
[ ] K点 ≥ 4个不可约K点（如果用了ISMEAR=-5）？
[ ] 用 ISMEAR=0 + SIGMA=0.01 和 ISMEAR=-5 对比验证过DOS？
[ ] 自旋极化体系 → 分通道检查过展宽效应？
[ ] 确认了SIGMA单位是eV（而非Ry/Hartree）？
[ ] 对于能量计算（吸附能等）→ 用了ISMEAR=-5的静态计算？
[ ] 半导体体系 → SIGMA < 带隙/2？
[ ] NCL/SOC → ISMEAR=-5？
```

## 总结

DOS的SMEAR展宽是那种"你觉得无所谓，审稿人觉得致命"的参数。它不像SCF不收敛那样明显报错——它会静默地给你一个"看起来合理"但物理错误的结果。

记住一句话: **SIGMA每大0.1eV，你就离真实电子结构远一步。当你画出来的DOS尖峰宽度≈SIGMA时，你画的就是高斯函数，不是态密度。**

关于ISMER=-5: 它是零展宽，等于是回到真实的物理态密度。但它被K点数诅咒：K点太稀=tetrahedron变成鬼=负DOS。你不确认就=闭着眼相信VASP。
