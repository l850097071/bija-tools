# 知乎45: DOS/能带结构分析——5个让你审稿翻车的隐藏雷区

**系列**: 避坑系列#12 | **字数**: ~2500 | **定位**: 催化/DFT计算化学垂直·真实经验

---

态密度(DOS)和能带结构是DFT计算里最"基础"的分析——基础到很多人觉得不需要检查。打开VASPKIT，跑一遍211和251，画出来贴论文里，完事。

就是这种"基础=没问题"的错觉，让DOS和能带成为审稿人被拒稿最频繁的翻车点。

我审过几篇投小NC的DFT稿子，DOS/能带分析几乎都有问题。不是图画得不好看，是你把错误的物理当正确的交上去了。审稿人如果认真看，一篇DOS图能挑出至少5个致命问题。

今天掰开讲。

## 雷区1: ISMEAR用错——你画的不是态密度，是展宽函数的形状

**死法**: SCF用ISMEAR=0(SIGMA=0.1)跑完收敛，直接拿同一套参数跑DOS。

**物理原因**: ISMEAR=0(Fermi-Dirac展宽)或ISMEAR=1(Methfessel-Paxton)引入了非物理的电子温度展宽。你用SIGMA=0.2eV跑SCF没问题(帮助收敛)，但用同样的SIGMA跑DOS——费米能级附近的态密度形状是**展宽函数的形状，不是真实的电子结构**。

**真实案例**: 一篇投NC的稿子，Co-d轨道在费米能级附近有个"尖峰"，作者解读为"Co活性位的高反应活性来源"。审稿人只看了一眼SIGMA=0.2——那个"尖峰"是Fermi-Dirac展宽边缘的人为截断。换成ISMEAR=-5(tetrahedron)重算，尖峰消失，变成正常的宽峰。

**操作铁律**:
```
SCF计算: ISMEAR=0, SIGMA=0.05-0.1  (帮助收敛，可以)
DOS计算: 
  金属体系  → ISMEAR=-5 (tetrahedron, 需≥4个不可约K点)
  绝缘体/半导体 → ISMEAR=0, SIGMA=0.01-0.05 (小展宽)
  自旋轨道耦合 → ISMEAR=0 (tetrahedron与SOC不兼容!)
  
# INCAR-DOS 关键设置
ICHARG = 11      # 读取CHGCAR, 非自洽
ISMEAR = -5      # 金属用tetrahedron
# ISMEAR = 0; SIGMA = 0.03  # 绝缘体/半导体用
LORBIT = 11      # 元素/轨道投影
NEDOS = 2000     # 能量分辨率
EMIN = -15; EMAX = 10  # 能量范围覆盖所需区间
```

## 雷区2: K点密度不够——你漏掉了关键特征

**死法**: 用SCF的K点网格(如3×3×1)直接跑DOS。图看起来"还行"，没有明显的锯齿。

**物理原因**: DOS计算需要**远高于SCF收敛**的K点密度。SCF收敛只需要总能量收敛到1meV/atom——3×3×1可能够了。但DOS的分辨率取决于布里渊区的采样密度：K点太稀→态密度被欠采样→窄峰丢失、能带交叉点被平滑掉。

**真实案例**: 一个C-N材料体系，作者用3×3×1的SCF K点跑了DOS，结论是"带隙中无缺陷态"。审稿人要求用9×9×1重算——带隙中出现了清晰的C-2p缺陷态，推翻了原文的"清洁带隙"论证。

**操作铁律**:
```
# K点密度规则
SCF:  3×3×1 (收敛总能量)
DOS:  9×9×1 ~ 15×15×1 (2-3倍SCF密度)
能带: 高对称路径 + 每段≥20个K点

# 收敛测试铁律
- 从3×3×1开始，翻倍递增(3→6→9→15)
- 直到DOS形状不再变化 → 那个K点密度才是可靠的
- 金属体系: 避免Γ-only, 必须用Monkhorst-Pack
- 六方体系: 同时跑奇数和偶数网格验证一致性
```

## 雷区3: NEDOS太小——你的费米能级附近是"马赛克"

**死法**: VASP默认NEDOS=301。跑完DOS用VASPKIT画图，费米能级附近看起来"还行"就过了。

**物理原因**: NEDOS=301意味着从EMIN到EMAX只有301个能量分格。如果你的能量跨度是25eV(如-15到10eV)，每个分格~0.083eV。这意味着任何宽度<0.1eV的特征(如窄d带峰、van Hove奇点)要么被完全抹平，要么变成锯齿状伪影。

**真实案例**: 一篇过渡金属SAC的稿子，Ni-d轨道的DOS在费米能级附近"平缓无特征"，作者解读为"Ni位惰性"。改用NEDOS=3000重算，d轨道分裂为清晰的t2g和eg双峰结构——完全推翻了"惰性"解读。审稿人的原话:"Your DOS resolution is insufficient to support the claim."

**操作铁律**:
```
NEDOS = 2000-3000  # 不是可选项，是必需品
EMIN/EMAX: 覆盖你关心的全部能量范围，留两侧各2-3eV余量

# 快速验证
grep "NEDOS" OUTCAR  # 确认实际使用的值
```

## 雷区4: PDOS > Total DOS——你的投影是"错觉"

**死法**: 用LORBIT=11做了元素投影，看到某个原子的PDOS大于总DOS，以为发现了什么"物理奇观"，在论文里大书特书。

**物理原因**: LORBIT=11使用的PAW投影函数定义在截断球(r=RWIGS or PAW sphere)内。这些球体**不填满整个计算胞**——球间区域(interstitial)的电子密度没有被任何原子"认领"。所以在某些能量区间，各原子"声称"的态密度之和可以大于总态密度，因为不同原子的投影球有重叠计数的可能性。

这**不是物理效应**，是PAW投影的数学artifact。审稿人看到这个，直接一个问题:"Please explain why PDOS exceeds total DOS."

**操作铁律**:
```
# 方案1: 接受投影的近似性
LORBIT = 11  # 最常用，但投影不严格归一化
# 论文中注明: "PDOS is an approximate decomposition within PAW spheres"

# 方案2: 用LORBIT=1 + 手动调RWIGS
LORBIT = 1
RWIGS = 1.2 1.0 0.8  # 各元素Wigner-Seitz半径，手动调
# 调RWIGS直到PDOS≈TDOS → 投影更可信

# 方案3: 交叉验证
# 用Quantum ESPRESSO或其他代码验证关键DOS结论
# 如PDOS>>TDOS的区间在另一代码中不出现 → artifact确认
```

## 雷区5: DOS对齐错误——你把两个体系的"零点"放在了不同位置上

**死法**: 跑了clean surface和surface+adsorbate两个体系，分别画DOS放在一张图上对比。"吸附后DOS红移了0.5eV——说明电荷转移！"结论掷地有声。

**物理原因**: VASP输出的本征能量以平均静电势为零点。不同体系(尤其是slab厚度不同、真空层厚度不同的体系)的平均静电势是不同的。把两个体系的费米能级简单对齐(E-EF=0放在同一点)来比较DOS——相当于把两个不同参照系的数据强行叠在一起。

真实的DOS位移可能是: (a)真实的电子结构变化、(b)真空能级差异造成的参照系偏移、(c)两者混合。你无法区分。

**真实案例**: 一篇表面催化的稿子，吸附O2后表面d带中心"下移0.3eV"，作者解读为"d带中心调控证据"。审稿人要求用真空能级或芯能级对齐——对齐后位移缩小到<0.05eV，d带调控论证完全坍塌。

**操作铁律**:
```
# 通用对齐方法
1. 芯能级对齐: 选一个远离表面的"旁观原子"(如slab底层原子)
   计算其1s能级在clean/adsorbed体系中的相对位置→以此对齐
   
2. 真空能级对齐: 提取LOCPOT→沿z方向平均→取真空区平坦段作对齐基准
   # VASP输出真空能级: grep "vacuum" LOCPOT
   
3. 审稿人铁问: "How did you align the energy scales?" 
   → 在方法部分明确写出对齐方法+验证收敛
```

## 3个附加检查(审稿人可能追问，但你不一定知道要检查)

### 附加1: LREAL设置一致性
SCF用了LREAL=Auto但DOS用了默认LREAL=.FALSE.→投影方式不同→PDOS不自洽。
**铁律**: SCF和DOS两轮LREAL设置完全一致。

### 附加2: ENCUT补充检查
POTCAR的ENMAX是"建议"值，做DOS需要更高精度。
**铁律**: ENCUT ≥ 1.3 × ENMAX(max)，跑收敛测试确认DOS无变化。

### 附加3: 自旋极化+轨道投影的符号
ISPIN=2时，LORBIT=11输出的PDOS有正负自旋。但某些后处理脚本会取绝对值→丢失自旋信息。
**铁律**: 直接读vasprun.xml的原始输出，不信任后处理脚本的"自动"处理。
