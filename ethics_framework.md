## 一、伦理风险定位

### 我们的服务定义
> "NC级逻辑链打磨——帮作者把论文从'结果堆砌'变成'有说服力的科学故事'"

### COPE红线对照

| COPE标准 | 我们的服务 | 风险评估 |
|-----------|-----------|---------|
| 不改变作者声音 | 逻辑流优化可能影响评价性语言 | ⚠️ 中 |
| 不生成实质论点 | 论证链重构=建议新论证结构 | 🔴 高 |
| 透明披露 | 未定义披露标准 | ⚠️ 中 |
| 作者最终控制权 | 未定义修改权限边界 | ⚠️ 中 |
| 不将AI列为作者 | ✅ 明确不列 | ✅ 低 |

## 二、三层服务边界定义

### L1: 语言辅助层（✅ 明确安全）
- 语法/拼写/标点修正
- 学术英语表达优化（中式英语→学术英语）
- 术语统一
- **不涉及**: 论证结构、逻辑流、结论方向
- **AI角色**: AI初稿→人工审核
- **披露**: "本文语言经AI辅助润色+人工校审"

### L2: 结构建议层（⚠️ 灰色地带）
- 段落顺序优化建议（标注为"建议"）
- 逻辑流可视化（展示当前论证链→标注薄弱环节）
- 审稿人视角预判（"这里审稿人可能问..."）
- **不涉及**: 直接重写论证、生成新内容
- **AI角色**: 分析工具（识别逻辑断裂）+ 人工判断（提出建议）
- **披露**: "本文接受了结构性编辑建议，所有修改由作者确认"

### L3: 禁止层（❌ 绝对不可）
- 生成新论证、新结论、新数据
- 改变论文核心论点的方向
- 以编辑判断替代作者科学判断
- 未标注的AI内容生成
- 冒充人类原创学术成果

## 三、操作协议

### 编辑流程
1. **接收稿件** → 作者签署伦理知情同意书
2. **AI分析** → 识别语法错误+逻辑断裂点（不改写）
3. **人工编辑** → 逐条标注修改类型（L1语法/L2建议）
4. **修改留痕** → Word修订模式，每条修改可追溯
5. **作者审核** → 作者逐条接受/拒绝，最后确认
6. **交付** → 含伦理声明模板 + 修改日志

### 作者知情同意（模板）
```
本研究使用了Bija NC写作润色服务。服务内容包括：
- L1: AI辅助语法润色 + 人工校审
- L2: 人工编辑提供的结构性建议（以修订模式标注）
所有L2修改均为建议性质，作者保留最终决定权。
论文的核心科学论点、数据和结论由作者独立完成。
```
