## A (Analyze) — 终极视频脚本模板

### 模板1: 「审稿人视角」系列 (25%, 最高收藏率)

```markdown
【0-3秒·反常识钩子】
"我审过X篇用{技术}做核心论据的稿子。其中Y篇的结果在{Z}后发生了定性反转。"

【4-15秒·焦虑建立】
"{具体审稿意见原文}——这是审稿人最常挑的{X}个角度之一。
如果你的论文刚好踩中这个坑，轻则大修，重则拒稿。"

【16-25秒·一个具体陷阱】
"比如{具体陷阱描述}。你做{操作}的时候，{错误的做法}会导致{后果}。
很多人的INCAR里根本就没有{关键参数}。"

【26-40秒·解法】
"解决方法很简单：{3步操作}。第一步{具体参数设置}，第二步{验证方法}，
第三步{在SI中放什么证据}。做完这三步，审稿人无话可说。"

【41-50秒·价值升华+CTA】
"{XXX}个投稿人中，只有{XX}%的人这三步全做对了。
你是那个会检查的，还是那个被审稿人挑出来的？
关注我，每期一个审稿人视角的避坑技巧。"
```

### 模板2: 「科研翻车实录」系列 (25%, 最高评论率)

```markdown
【0-3秒·故事钩子】
"我导师说我的计算有问题，但他也不知道问题在哪。
我查了三天，发现是{参数}设错了。"

【4-15秒·共鸣建立】
"做计算的都有过这种经历：代码跑了三个星期，结果出来了，
但直觉告诉你——这数字不对。然后你开始怀疑自己。"

【16-30秒·翻车过程】
"回到我的翻车现场。我在算{体系}的{性质}，
跑出来的{结果}是{数值}。文献里明明是{文献值}。
我检查了{逐项排查}——都没问题。"

【31-45秒·发现根因】
"直到第四天，我发现{根因}。因为{物理原理}，
这个参数其实应该设成{正确值}而不是{错误值}。
改了之后，结果从{错误值}变成{正确值}。"

【46-55秒·教训+CTA】
"这个坑我替你踩过了。记住：{一句话教训}。
评论区说说，你踩过什么让你查到崩溃的坑？"
```

### 模板3: 「避坑清单」系列 (20%, 最快生产)

```markdown
【0-3秒·利益直给钩子】
"{技术}分析最容易出错的{X}个地方。
收藏这个视频，投稿前一个个检查。"

【4-30秒·快速罗列】
"第1：{陷阱1}，检查{参数1}是否设成{值1}。
第2：{陷阱2}，确认{操作2}之前做了{检查2}。
第3：{陷阱3}，{症状}说明{问题}，改成{修复}。
第4：{陷阱4}，..."
（每陷阱5-8秒，一口气讲完5-7个）

【31-40秒·CTA】
"完整检查单在评论区。关注我，每天一个避坑视频。"
```
